# NAVI-IMS
## Integrated Management System
### ระบบจัดการแบบบูรณาการ

---

## ปัญหาที่เราแก้

| ปัญหาเดิม | NAVI-IMS แก้ได้ |
|-----------|----------------|
| ข้อมูลกระจาย หลายระบบ ไม่เชื่อมกัน | **ศูนย์กลางเดียว** — Odoo เก็บทุกอย่าง |
| ไม่รู้สถานการณ์ real-time | **แผนที่ + Live Video** แบบ real-time |
| แจ้งเตือนช้า ต้องรอคนรายงาน | **AI ตรวจจับอัตโนมัติ** + แจ้ง LINE/Slack ทันที |
| ไม่รู้ว่าอุปกรณ์พร้อมใช้ไหม | **ระบบซ่อมบำรุง** + ติดตามสถานะ |
| เข้า-ออก ตรวจด้วยคน | **AI จดจำคน/รถ/เรือ** → เปิดประตูอัตโนมัติ |
| รายงานผู้บริหาร ทำเอง | **Executive Dashboard** KPI อัตโนมัติ |

## สถาปัตยกรรมระบบ

```
                    ┌─────────────────────┐
                    │   Odoo 19           │
                    │   ศูนย์กลางข้อมูล    │
                    │   ทั้งหมดอยู่ที่นี่    │
                    └──────────┬──────────┘
                               │
          ┌────────┬───────────┼───────────┬────────┐
          ▼        ▼           ▼           ▼        ▼
      MediaMTX   Node.js    Celery     Inngest   PostgreSQL
      (Video)    (GPS)      (AI)       (Workflow) (Database)
```

- **Odoo 19** = ศูนย์กลาง ทุกข้อมูลอยู่ที่นี่
- **Service อื่นๆ** = เครื่องมือช่วย ไม่มีข้อมูลเป็นของตัวเอง
- **Open Source ทั้งหมด** ไม่มีค่า license

## 7 Modules — เลือกติดตั้งได้

### Core (บังคับ)

**patrol_command** — ศูนย์บัญชาการ

| ความสามารถ | รายละเอียด |
|-----------|-----------|
| โครงสร้างหน่วย | กองร้อย → หมวด → หมู่ → ชุด |
| กำลังพล | ทะเบียน + ยศ + สังกัด + Odoo login |
| อุปกรณ์ | กล้อง/drone/body cam + สั่งเปิด-ปิดจาก Odoo |
| ภารกิจ | มอบหมายคน+อุปกรณ์ → เริ่ม → เสร็จ |
| GPS Tracking | real-time + ดูย้อนหลัง |
| Live Video | ดูจากกล้องทุกตัวผ่าน browser |
| เหตุการณ์/SOS | สร้าง → มอบหมาย → แก้ไข → ปิด |
| ซ่อมบำรุง | แจ้งซ่อม + แผนซ่อมตามกำหนด |
| แจ้งเตือน | LINE Notify / Slack / Discord |
| Executive Dashboard | KPI สำหรับผู้บริหาร |

### Optional (เลือกติดตั้งตามต้องการ)

| Module | ความสามารถ |
|--------|-----------|
| **patrol_personnel** | ประวัติฝึก, ตารางเวร, สุขภาพ+ความพร้อม |
| **patrol_inventory** | คลังพัสดุ (อาวุธ/กระสุน/เสบียง), เบิก-จ่าย+อนุมัติ |
| **patrol_intelligence** | Watchlist (คน/รถ/เรือ), รายงานข่าวกรอง, พื้นที่เสี่ยง |
| **patrol_geofence** | กำหนดเขต, แจ้งเตือนเข้า-ออก, สร้าง incident อัตโนมัติ |
| **patrol_access** | ทะเบียนคน/รถ/เรือ, ประตู/ท่าเรือ, เปิดอัตโนมัติ, สอดส่อง |
| **patrol_geolocation** | พิกัดจริงจากกล้อง, Camera Calibration, ติดตามข้ามกล้อง |

## กล้อง 4 จุดใช้งาน

| จุดติดตั้ง | AI ตรวจจับ | Action |
|-----------|-----------|--------|
| **ประตูทางเข้า** | คน/รถ → match ทะเบียน | เปิดอัตโนมัติ / รออนุมัติ / บล็อก |
| **ในพื้นที่** | คน/รถ → สอดส่อง | บันทึก + แจ้งเตือนถ้าตรง watchlist |
| **ปากร่องน้ำ** | เรือ → match ทะเบียน | บันทึก + แจ้งเตือนเรือไม่รู้จัก |
| **ท่าเรือ** | เรือ/คน → match | บันทึก + ควบคุมจุดขึ้น-ลง |

### Full Loop (กล้อง Fixed 24/7)

```
กล้อง → AI ตรวจจับ → รู้จัก? ─── ใช่ → บันทึก (เปิดประตูถ้ามี)
                        │
                        ไม่
                        │
                 ตรง Watchlist? ─── ใช่ → 🚨 สร้าง Incident + แจ้ง LINE
                        │
                        ไม่
                        │
                   บันทึก + ตรวจกฎแจ้งเตือน
```

## ภารกิจ (Mission)

```
ผบ. สร้างภารกิจใน Odoo
  │
  ├── มอบหมาย ทีมลาดตะเวน + body cam
  ├── มอบหมาย ทีม drone
  ├── กำหนดพื้นที่ปฏิบัติการ (GeoJSON)
  │
  ▼ กด "เริ่มปฏิบัติการ"
  │
  ├── เริ่ม stream กล้องทั้งหมด อัตโนมัติ
  ├── แจ้งทุกทีมผ่าน LINE/Slack
  ├── GPS tracking เริ่มบันทึก
  ├── AI เฝ้าระวังทุกกล้อง
  │
  ▼ ระหว่างภารกิจ
  │
  ├── เหตุการณ์ → สร้าง Incident อัตโนมัติ
  ├── SOS → แจ้ง ผบ.หมู่ ทันที
  │     └── ไม่ตอบ 15 นาที → escalate ผบ.หมวด
  │           └── ไม่ตอบ → escalate ผบ.กองร้อย
  │
  ▼ กด "เสร็จสิ้น"
  │
  ├── หยุด stream ทั้งหมด
  └── สรุปผล: เหตุการณ์ทั้งหมด, เวลา, GPS tracks
```

## แจ้งเตือน

| ช่องทาง | ตั้งค่าใน Odoo | ตัวอย่าง |
|---------|-------------|---------|
| **LINE Notify** | ใส่ token | 🚨 SOS จาก Alpha-1 ที่ 13.75, 100.50 |
| **Slack** | ใส่ webhook URL | ⚠️ AI ตรวจพบเรือไม่รู้จักที่ปากร่องน้ำ |
| **Discord** | ใส่ webhook URL | 📋 ภารกิจ MSN-2026-0042 เริ่มแล้ว |
| **Odoo** | อัตโนมัติ | notification ใน browser |

กรองได้:
- ตาม **severity** (วิกฤต/สูง/กลาง/ต่ำ)
- ตาม **ประเภท** (SOS/Incident/Mission/Geofence/Access)

## ระบบสิทธิ์ตามยศ

```
ผบ.กองร้อย ──── ดูทุกอย่าง + ตั้งค่า
    │
ผบ.หมวด ─────── สร้างภารกิจ + จัดการอุปกรณ์
    │
ผบ.หมู่ ──────── ดูทหารในหมู่ + จัดการเหตุการณ์
    │
ทหาร ─────────── ดูของตัวเอง + แจ้งซ่อม + SOS
```

ทหารทุกคน login เข้า Odoo ได้ — เห็นข้อมูลตามสิทธิ์

## Geolocation — พิกัดจริงจากกล้อง

| วิธี | ความแม่น | เงื่อนไข |
|------|---------|---------|
| **GPS Fusion** | ±5m | AI match ทหารที่มี GPS จากมือถือ |
| **Camera Calibration** | ±5-20m | ตั้ง calibration กล้อง (ทิศ, FOV, ความสูง) |
| **Homography** | ±3-10m | จิ้ม 4 จุดบนภาพ + กำหนด GPS |
| **Fallback** | ±50m | ใช้พิกัดกล้อง (ไม่ต้องตั้งอะไร) |

ระบบเลือกวิธีที่ดีที่สุดอัตโนมัติ

## Scalability

| คำถาม | คำตอบ |
|-------|-------|
| รองรับกี่กล้อง? | ไม่จำกัด — เพิ่ม Celery worker |
| รองรับกี่คน? | ไม่จำกัด — PostgreSQL scale ได้ |
| AI ช้าไหม? | เพิ่ม worker / เพิ่ม GPU |
| Video กี่ stream? | ขึ้นกับ bandwidth (MediaMTX scale ได้) |
| รองรับหลายฐาน? | ได้ — แยก Odoo database per site |

## Demo

### Codespaces (ทดลองได้ทันที)

1. เปิด https://github.com/monthop-gmail/navi-ims
2. กดปุ่ม **Code** → **Codespaces** → **Create**
3. รอ container สร้าง (~1 นาที) → build อัตโนมัติ (~3-5 นาที)
4. ไปที่ tab **PORTS** → คลิก globe icon ของ port 8069
5. Login: **admin / admin**

### Local Docker

```bash
git clone https://github.com/monthop-gmail/navi-ims.git
cd navi-ims
cp .env.example .env
docker compose up -d --build    # แค่นี้ — Odoo auto-init ทุก module
```
