# NAVI-IMS — คู่มือทดสอบ

## เตรียมตัว

### Codespaces

1. เปิด https://github.com/monthop-gmail/navi-ims
2. **Code** → **Codespaces** → **Create codespace on master**
3. รอ container สร้าง (~1 นาที) → Terminal จะรัน `docker compose up -d --build` อัตโนมัติ (~3-5 นาที)
4. Odoo auto-init DB + ลง 7 modules (~30 วินาที)
5. ไปที่ tab **PORTS** ด้านล่าง → คลิก globe icon ของ port **8069**
6. Login: **admin / admin**

> ใน Codespaces ไม่ใช้ `localhost` — ดู URL จาก tab PORTS

### Local Docker

```bash
git clone https://github.com/monthop-gmail/navi-ims.git
cd navi-ims
cp .env.example .env

# Start ทุกอย่าง (Odoo จะ auto-init DB + ลง 7 modules เอง)
docker compose up -d --build
# รอ ~30 วินาที แล้วเปิด
```

เปิด http://localhost:8069 → Login: **admin / admin**

## ทดสอบละเอียด (Scenario Test)

### Scenario 1: ทหารลาดตะเวน + GPS

**ต้องใช้ 2 คน** (หรือ 2 หน้าจอ)

**คนที่ 1 — มือถือ:**
1. เปิด soldier.html (port 3000 → `/patrol/soldier.html`)
2. ใส่ callsign `Alpha-1` → กด **เริ่มลาดตะเวน**
3. อนุญาต GPS + กล้อง
4. เดินไปมา → ดูว่า GPS อัพเดทบนหน้าจอ

**คนที่ 2 — PC:**
1. เปิด Odoo → เมนู **ปฏิบัติการ** → **ศูนย์บัญชาการ**
2. เห็น marker Alpha-1 บนแผนที่ (จุดสีเขียว)
3. รอ 5 วินาที → marker ขยับตามที่คนที่ 1 เดิน
4. กดที่ marker → เห็น popup ชื่อ + สถานะ
5. กดปุ่ม 📍 ใน sidebar → เห็นเส้นทางที่เดิน (polyline สีเขียว)

**ผ่านเมื่อ:**
- ☐ marker ขยับ real-time (อาจ delay 2-5 วินาที)
- ☐ กดแสดงเส้นทาง → เห็น polyline
- ☐ เมนู GPS Logs → เห็นข้อมูล lat/lng บันทึก

### Scenario 2: SOS ฉุกเฉิน

**ต่อจาก Scenario 1**

**คนที่ 1 — มือถือ:**
1. กำลังลาดตะเวนอยู่
2. **กดค้าง** ปุ่ม SOS **1 วินาที** → ปุ่มเป็นสีแดงกระพริบ
3. ปล่อยปุ่ม → SOS ยกเลิก

**คนที่ 2 — PC:**
1. Command Center → เห็น marker แดงกระพริบ ที่ตำแหน่ง SOS
2. เมนู **ปฏิบัติการ** → **เหตุการณ์** → เห็น incident ใหม่:
   - ชื่อ: "SOS — Alpha-1 (...)"
   - ประเภท: SOS ฉุกเฉิน
   - ความรุนแรง: วิกฤต
3. ลอง workflow:
   - กด **มอบหมาย** (เลือกผู้รับผิดชอบก่อน)
   - กด **เริ่มดำเนินการ**
   - กด **แก้ไขแล้ว**
   - กด **ปิด**

**ผ่านเมื่อ:**
- ☐ incident สร้างอัตโนมัติเมื่อกด SOS
- ☐ ประเภท = SOS, ความรุนแรง = วิกฤต
- ☐ workflow เปลี่ยน state ครบ: ใหม่ → มอบหมาย → ดำเนินการ → แก้ไข → ปิด

### Scenario 3: ภารกิจเต็ม lifecycle

1. เมนู **ปฏิบัติการ** → **ภารกิจ** → สร้างใหม่
   - ชื่อ: "ลาดตะเวนรอบเช้า"
   - ประเภท: ลาดตะเวน
   - ผู้บังคับภารกิจ: เลือกทหาร
   - กำลังพล: เลือก Alpha-1, Alpha-2
   - อุปกรณ์: เลือก BCAM-Alpha-1

2. กด **วางแผน** → state เป็น "วางแผน"

3. กด **เริ่มปฏิบัติการ** → state เป็น "ปฏิบัติการ"

4. ระหว่างภารกิจ:
   - ทหารเปิด soldier.html → GPS + video ส่งเข้า Odoo
   - ลองกด SOS → incident สร้างภายใต้ภารกิจนี้

5. กด **เสร็จสิ้น** → state เป็น "เสร็จสิ้น"

**ผ่านเมื่อ:**
- ☐ state เปลี่ยน: ร่าง → วางแผน → ปฏิบัติการ → เสร็จสิ้น
- ☐ กำลังพล + อุปกรณ์ แสดงถูกต้อง
- ☐ incident ที่เกิดระหว่างภารกิจ ผูกกับภารกิจนี้
- ☐ เลขที่ภารกิจ auto-generate (MSN-2026-xxxx)

### Scenario 4: ซ่อมบำรุง

1. เมนู **ทรัพยากร** → **อุปกรณ์** → เลือก CAM-FIXED-01

2. ดูส่วน "ซ่อมบำรุง" → เพิ่มแผนซ่อม:
   - ชื่อ: "ทำความสะอาดเลนส์"
   - ทุก 14 วัน

3. กดปุ่ม 🔧 สร้างใบแจ้งซ่อม → ระบบสร้างอัตโนมัติ

4. เมนู **ซ่อมบำรุง** → **ใบแจ้งซ่อม** → เห็น MNT-2026-xxxx

5. Workflow:
   - เลือกช่าง (ช่องช่างผู้รับผิดชอบ) → กด **มอบหมายช่าง**
   - กด **เริ่มซ่อม**
   - กด **ซ่อมเสร็จ**
   - กด **ตรวจรับ**

6. กลับไปดู อุปกรณ์ CAM-FIXED-01 → สถานะกลับเป็น "พร้อม"

**ผ่านเมื่อ:**
- ☐ สร้างใบแจ้งซ่อมจากแผนได้
- ☐ workflow: แจ้งซ่อม → มอบหมาย → ซ่อม → ตรวจรับ
- ☐ อุปกรณ์เปลี่ยน maintenance → ready อัตโนมัติ
- ☐ เลขที่ MNT-2026-xxxx

### Scenario 5: ข้อมูลเข้า-ออก + เรือ

1. เมนู **เข้า-ออก** → **ทะเบียนบุคคล** → สร้างใหม่
   - ชื่อ: ทดสอบ
   - ประเภท: ผู้มาติดต่อ
   - สิทธิ์: เฉพาะที่กำหนด
   - อัพโหลดรูปถ่าย

2. เมนู **เข้า-ออก** → **ทะเบียนยานพาหนะ** → สร้างใหม่
   - หมวด: **ทางน้ำ**
   - ประเภท: เรือประมง
   - ทะเบียน: "สมบูรณ์ 5"
   - ใส่: ชื่อเรือ, ความยาว, ท่าเรือประจำ

3. เมนู **เข้า-ออก** → **ประตู / จุดเข้า-ออก** → สร้างใหม่
   - ชื่อ: ท่าเรือ 1
   - ประเภท: **ท่าเรือ**
   - เปิดอัตโนมัติเมื่อรู้จัก: ✅
   - ต้องอนุมัติเมื่อไม่รู้จัก: ✅

**ผ่านเมื่อ:**
- ☐ สร้างทะเบียนบุคคล + รูปถ่ายได้
- ☐ เลือกหมวด "ทางน้ำ" → เห็นประเภทเรือ (ประมง, บรรทุก, เร็ว, หางยาว ฯลฯ)
- ☐ กรอกข้อมูลเรือ (ชื่อเรือ, ความยาว, AIS MMSI) ได้
- ☐ สร้างจุดเข้า-ออกประเภท "ท่าเรือ" / "ปากร่องน้ำ" ได้

### Scenario 8: Security (สิทธิ์ตามยศ)

**ตั้งค่า:**
1. เมนู **ตั้งค่า** → **ผู้ใช้** → สร้าง 2 users:
   - `soldier1` (email: soldier1@test.com) → กลุ่ม: **ทหาร**
   - `leader1` (email: leader1@test.com) → กลุ่ม: **ผบ.หมู่**

2. เมนู **ทรัพยากร** → **กำลังพล**:
   - Alpha-1 → ตั้ง Odoo User = soldier1
   - Alpha-Lead → ตั้ง Odoo User = leader1

**ทดสอบ:**
3. Logout → Login ด้วย soldier1:
   - เห็นเฉพาะข้อมูลตัวเอง
   - สร้าง incident ได้ / แจ้งซ่อมได้

4. Logout → Login ด้วย leader1:
   - เห็นทหารในหมู่ (Alpha-1, Alpha-2)
   - จัดการ incident ได้ (แก้ไข + มอบหมาย)

**ผ่านเมื่อ:**
- ☐ soldier1 เห็นเฉพาะของตัวเอง
- ☐ leader1 เห็นทหารในหมู่
- ☐ soldier1 แก้ไขข้อมูลทหารคนอื่นไม่ได้

### Scenario 10: คลังพัสดุ

1. เมนู **คลังพัสดุ** → **รายการพัสดุ** → สร้างใหม่
   - ชื่อ: "กระสุน 5.56mm"
   - ประเภท: กระสุน
   - จำนวนคงเหลือ: 1000
   - จำนวนขั้นต่ำ: 200

2. เมนู **คลังพัสดุ** → **ใบเบิกพัสดุ** → สร้างใหม่
   - ประเภท: เบิก
   - เพิ่มรายการ: กระสุน 5.56mm จำนวน 100
   - กด **ส่งขออนุมัติ**
   - กด **อนุมัติ**
   - กด **เบิกจ่าย**

3. กลับไปดู รายการพัสดุ → กระสุน 5.56mm → คงเหลือ 900

**ผ่านเมื่อ:**
- ☐ สร้างพัสดุ + ตั้ง stock ขั้นต่ำได้
- ☐ ใบเบิก workflow: ร่าง → ส่งขอ → อนุมัติ → เบิกจ่าย
- ☐ จำนวนคงเหลือลดอัตโนมัติ (1000 → 900)
